# 简历优化确认接口文档

> 前端页面：`src/views/resume/ResumeOptimizeView.vue`  
> Base URL：`/api`

## 1. AI 补全预览

`POST /resumes/{resume_id}/optimizations/{optimization_id}/confirmations/ai-preview`

请求：

```json
{
  "optimized_content": "当前优化后简历全文",
  "confirmation_questions": ["需要确认的问题"],
  "confirmed_answers": [
    {
      "question": "用户正在确认的具体问题",
      "answer": "用户填写的真实确认内容，例如项目链接、上线情况、量化成果"
    }
  ],
  "feedback": "用户不满意后的修改意见，可选"
}
```

说明：

- `confirmed_answers` 是本次新增字段，用于逐条确认问题。
- 用户在前端对某个问题填写真实信息并点击“确认并让 AI 补全”时，前端只传该问题和对应答案。
- 后端应基于 `confirmed_answers.answer` 修改 `optimized_content`，不要仅凭问题本身编造内容。
- 如果确认内容已成功写入简历，对应问题应进入 `resolved_questions`，并从 `remaining_questions` 中移除。

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "preview_id": "可选",
    "added_content": "AI 本次新增/改写的内容，必须给用户看",
    "optimized_content": "合并补全后的简历全文",
    "summary": "本次 AI 做了什么",
    "resolved_questions": ["已解决的问题"],
    "remaining_questions": ["仍需确认的问题"],
    "change_items": []
  }
}
```

说明：前端不会自动采用，用户点击“采用 AI 补全内容”后才会更新页面里的优化后简历。

## 2. 手动补充预览

`POST /resumes/{resume_id}/optimizations/{optimization_id}/confirmations/manual-preview`

请求：

```json
{
  "optimized_content": "当前优化后简历全文",
  "confirmations": [
    {
      "question": "需要确认的问题",
      "answer": "用户手动填写的真实内容"
    }
  ]
}
```

响应同 AI 补全预览：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "added_content": "手动信息整合后新增/改写的内容，必须给用户看",
    "optimized_content": "合并手动补充后的简历全文",
    "summary": "本次手动补充做了什么",
    "resolved_questions": [],
    "remaining_questions": [],
    "change_items": []
  }
}
```

## 3. 忽略确认信息

`POST /resumes/{resume_id}/optimizations/{optimization_id}/confirmations/dismiss`

请求：

```json
{
  "confirmation_questions": ["用户选择不需要的信息"]
}
```

响应返回完整优化结果：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 1,
    "optimized_content": "当前优化后简历全文",
    "optimization_summary": "已忽略待确认信息",
    "change_items": [],
    "confirmation_questions": []
  }
}
```

## 4. 保存最终优化简历

`POST /resumes/{resume_id}/optimizations/{optimization_id}/save`

请求：

```json
{
  "optimized_content": "用户保存前确认/微调后的最终简历全文",
  "confirmation_actions": [
    {
      "type": "ai",
      "title": "AI 补全到简历",
      "questions": ["处理过的问题"],
      "added_content": "用户采用的新增内容",
      "summary": "处理说明",
      "feedback": "用户给 AI 的修改意见，可选",
      "created_at": "2026/7/16 10:00:00"
    }
  ],
  "change_items": []
}
```

响应返回保存后的优化结果：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 1,
    "resume_id": 7,
    "title": "张三-优化简历",
    "is_saved": true,
    "saved_at": "2026-07-16T10:00:00",
    "original": "原始简历",
    "optimized_content": "最终保存的优化简历全文",
    "optimization_summary": "保存成功",
    "change_items": [],
    "confirmation_questions": []
  }
}
```

## 5. 前端按钮对应

| 按钮 | 接口 |
| --- | --- |
| AI 补全到简历 | `ai-preview` |
| 不满意，告诉 AI 怎么改 | `ai-preview`，带 `feedback` |
| 手动填写后加入简历 | `manual-preview` |
| 暂不需要这些信息 | `dismiss` |
| 保存更改内容 / 确认保存 | `save` |
