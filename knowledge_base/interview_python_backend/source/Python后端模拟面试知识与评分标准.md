# Python 后端模拟面试知识与评分标准

## 面试覆盖范围

Python 后端岗位应覆盖 Python 语言基础、异步编程、FastAPI、Pydantic、SQLAlchemy、MySQL、Redis、RESTful API、鉴权、安全、测试、性能和部署。题目必须结合候选人简历和岗位描述，不能把知识库内容冒充为候选人的真实经历。

## Python 与异步编程

高质量回答应能解释可变与不可变对象、迭代器与生成器、上下文管理器、装饰器、异常边界、类型标注，以及 asyncio 事件循环、协程、Task、并发限制和阻塞调用隔离。考察异步时应追问：为何不能在 async 路由里直接运行阻塞 I/O，如何使用线程池或异步客户端，如何处理取消、超时和资源关闭。

## FastAPI 与接口设计

重点包括依赖注入、请求校验、响应模型、中间件、异常处理、OpenAPI、幂等性、分页、状态码和版本管理。优秀回答应说明业务校验与 Pydantic 结构校验的边界，能够设计一致的错误响应，并说明如何避免在响应中泄漏内部异常或敏感字段。

## SQLAlchemy 与 MySQL

重点包括事务边界、连接池、索引、慢查询、N+1、隔离级别、死锁和迁移。项目题应追问：如何保证投递记录幂等；如何用唯一约束加事务处理并发；为何不能只依赖“先查询再插入”；如何通过 Alembic 安全发布表结构变更。

## Redis 与缓存

考察缓存穿透、击穿、雪崩、过期策略、分布式锁、幂等键和消息去重。高分回答需明确缓存只是派生数据，说明一致性策略、锁超时和兜底方案，避免把 Redis 锁描述成绝对可靠的事务替代品。

## 安全、测试与可观测性

安全题包含 JWT 生命周期、刷新令牌、密码哈希、权限校验、上传文件校验、密钥管理和日志脱敏。测试应包含单元测试、接口集成测试、数据库状态流转、外部 AI/爬虫 Fake、失败回写。可观测性应包含结构化日志、请求 ID、延迟、错误率、模型调用与 Token 审计。

## 项目与行为题

项目题采用 STAR 或“背景—职责—方案—取舍—结果—复盘”，只根据简历已知事实提问；涉及结果数字时必须要求候选人自行确认。行为题考察协作、需求变化、事故处理、优先级与复盘，评价重点是具体行为和决策依据。

## 百分制评分锚点

- 90–100：准确、完整、有工程取舍与真实例子，能覆盖异常、并发、安全或验证方式。
- 75–89：核心正确且结构清晰，有部分实践细节，但边界条件或取舍不够完整。
- 60–74：理解基本概念，答案较泛，缺少可执行步骤或证据。
- 40–59：仅有零散关键词，存在重要遗漏或混淆。
- 0–39：答非所问、核心错误，或无法给出有效回答。

维度统一采用：专业知识、问题分析、实践证据、沟通结构。不得因表达风格、口音或与岗位无关的个人特征扣分。

## 针对性题库生成

报告题库必须对应本次薄弱项。每个薄弱项至少生成一道题，并附参考要点。不得直接复用本场原题；难度应在候选人当前水平附近逐步提高。
