# 职业规划提前推进与阶段考核接口

> 本文只记录本次前端新增功能需要的后端接口。全部接口需要 JWT 登录态，并按当前用户校验执行计划、任务和考核归属。

## 一、执行计划状态扩展

`GET /api/career-plan-executions/current` 返回的 `data` 增加：

```json
{
  "today_completed": true,
  "can_advance": true,
  "next_task_available": true,
  "ahead_task_count": 2,
  "ahead_days": 1,
  "stage_progress": 100,
  "assessment_ready": true,
  "current_stage_status": "ready_for_assessment",
  "active_assessment_id": null
}
```

阶段状态：`in_progress`、`ready_for_assessment`、`assessing`、`passed`、`needs_improvement`。

任务对象可增加：

```json
{
  "is_advanced": true,
  "original_planned_date": "2026-07-18",
  "advanced_at": "2026-07-16T16:00:00"
}
```

以上状态必须由后端计算，前端只展示。

## 二、提前解锁下一项任务

`POST /api/career-plan-executions/{execution_plan_id}/advance`

处理要求：

- 只有当天必做任务全部完成后才允许提前执行。
- 按阶段、周次和任务顺序选择下一项未开始任务，不重复创建任务。
- 保留原计划日期，标记 `is_advanced=true`。
- 建议限制每天最多提前 3 项，超过时返回明确提示。

返回：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "overview": {},
    "advanced_task": {
      "id": 705,
      "execution_plan_id": 601,
      "title": "完成依赖注入练习",
      "task_type": "practice",
      "stage": "FastAPI 基础",
      "week_no": 1,
      "planned_date": "2026-07-16",
      "original_planned_date": "2026-07-18",
      "is_advanced": true,
      "is_required": true,
      "status": "pending"
    }
  }
}
```

`overview` 返回完整最新 `CareerExecutionOverview`。

## 三、生成阶段考核

`POST /api/career-plan-executions/{execution_plan_id}/assessments`

仅当 `assessment_ready=true` 时允许。重复请求应复用尚未提交的考核，避免重复调用模型。

Agent 出题上下文包括：

- 当前阶段任务及验收标准。
- 用户完成状态和打卡备注。
- 用户在本阶段提交的学习问题及 AI 回答。
- 项目实践和阶段产出。
- 职业规划知识库的必要召回内容。

禁止把密码、Token、API Key 等敏感信息放进题目。

返回：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "task_id": "task_assessment_generate_xxx",
    "status": "pending",
    "result_id": null,
    "assessment_id": 901,
    "poll_after_seconds": 1
  }
}
```

通用 AI Task 增加 `task_type = career_stage_assessment`，成功后 `result_id=assessment_id`。

## 四、获取考核题目

`GET /api/career-plan-executions/assessments/{assessment_id}`

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 901,
    "execution_plan_id": 601,
    "stage": "FastAPI 基础与接口开发",
    "status": "ready",
    "passing_score": 70,
    "time_limit_minutes": null,
    "created_at": "2026-07-16T16:10:00",
    "questions": [
      {
        "id": 1001,
        "question_type": "single_choice",
        "title": "FastAPI 中 Depends 的主要作用是什么？",
        "description": null,
        "options": [
          { "key": "A", "label": "声明和复用依赖" },
          { "key": "B", "label": "创建数据库表" }
        ],
        "points": 10,
        "code_language": null
      }
    ]
  }
}
```

题型：`single_choice`、`multiple_choice`、`short_answer`、`code`。建议总题量 6~10 题。

## 五、提交答案并创建 AI 评分任务

`POST /api/career-plan-executions/assessments/{assessment_id}/submit`

请求：

```json
{
  "answers": [
    { "question_id": 1001, "answer": "A" },
    { "question_id": 1002, "answer": ["A", "C"] },
    { "question_id": 1003, "answer": "依赖注入可以……" }
  ]
}
```

处理要求：考核只能提交一次；客观题按确定规则评分，简答题和代码题由 Agent 结合评分标准评价并给出依据。

返回 AI Task，增加 `task_type = career_stage_assessment_evaluation`：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "task_id": "task_assessment_evaluate_xxx",
    "status": "pending",
    "result_id": null,
    "assessment_id": 901,
    "poll_after_seconds": 1
  }
}
```

## 六、获取考核结果

`GET /api/career-plan-executions/assessments/{assessment_id}/result`

返回字段：

- `score`、`passing_score`、`passed`、`summary`。
- `strengths`、`weaknesses`、`improvement_advice`。
- `question_results`：题目 ID、得分、满分、用户答案、参考答案和 AI 点评。
- `remediation_available`：未通过时能否生成补强任务。
- `status`：`passed` 或 `needs_improvement`。

## 七、考核后的计划流转

考核通过：

`POST /api/career-plan-executions/assessments/{assessment_id}/next-stage`

- 把当前阶段标记为通过。
- 激活下一阶段任务。
- 返回完整最新 `CareerExecutionOverview`。
- 必须幂等，重复点击不能重复生成任务。

考核未通过：

`POST /api/career-plan-executions/assessments/{assessment_id}/remediation`

- 根据错题、薄弱点、打卡和问答记录生成补强任务。
- 原有完成记录和进度不能被清空。
- 补强任务完成后重新开放阶段考核。
- 返回完整最新 `CareerExecutionOverview`。

## 安全与一致性

- 所有资源必须属于当前用户，越权统一返回 404。
- 代码答案只能作为文本交给受控 Agent 分析，API 服务不得直接执行用户代码。
- 模型评分必须保存评分依据，不能只保存总分。
- 任务完成、提前推进、考核生成、提交和阶段流转都必须有后端幂等保护。
