# 职业生涯规划前端接口文档

> 更新时间：2026-07-15  
> 模块路径：`src/views/career/CareerPlanView.vue`  
> 当前结论：后端暂未实现职业生涯规划专用接口，前端已改为本地生成规划结果，避免页面请求不存在接口。

## 1. 当前可用状态

职业生涯规划页面当前可以独立使用：

- 用户填写教育背景、工作年限、技能、工作经历、学习时间、期望岗位和项目经历。
- 技能支持下拉选择和自定义输入。
- 项目附件当前只在前端本地记录文件名、大小、类型，不上传后端。
- 点击“生成职业生涯规划”后，前端根据表单内容生成职业画像、岗位推荐、能力差距、学习路径、行动计划和风险建议。

当前不会调用职业规划后端接口，因此不会再因为接口不存在导致生成失败。

## 2. 后端现状检查

当前后端已注册路由包括：

- `/api/resumes`
- `/api/resumes/{resume_id}/optimizations`
- `/api/jobs`
- `/api/job-platforms`
- `/api/ai/tasks/{task_id}`
- 用户登录、注册、刷新、登出、用户信息

当前后端未注册以下职业规划接口：

- `POST /api/career-planning/profiles`
- `POST /api/career-plans`
- `GET /api/career-plans/{plan_id}`
- `POST /api/career-plans/project-files/upload`
- `DELETE /api/career-plans/project-files/{file_id}`

因此这些接口在真实后端模式下属于空接口/未实现接口。

## 3. 前端保留的接口封装

文件：`src/api/career.ts`

这些封装仍然保留，作为后端未来实现后的对接入口：

| 方法 | 请求 | 说明 | 当前页面是否调用 |
| --- | --- | --- | --- |
| `createProfile` | `POST /career-planning/profiles` | 创建职业规划用户画像 | 否 |
| `uploadProjectFile` | `POST /career-plans/project-files/upload` | 上传项目附件 | 否 |
| `deleteProjectFile` | `DELETE /career-plans/project-files/{fileId}` | 删除项目附件 | 否 |
| `createPlan` | `POST /career-plans` | 创建职业规划生成任务 | 否 |
| `getPlan` | `GET /career-plans/{planId}` | 获取职业规划结果 | 否 |

说明：axios 基础路径是 `/api`，所以封装里的 `/career-plans` 实际请求地址是 `/api/career-plans`。

## 4. 当前前端数据流

当前页面不依赖后端，生成流程如下：

1. 校验表单必填项：教育背景、工作年限、技能标签、工作经历、每周学习时间。
2. 校验项目经历：如果填写了项目任意字段，则项目名称、描述、角色必须完整。
3. 附件选择后生成本地附件对象：
   - `id`
   - `original_filename`
   - `file_type`
   - `file_size`
   - `status: completed`
4. 点击生成后根据表单内容推断主岗位方向：
   - 已填写期望岗位时，优先使用期望岗位。
   - 技能包含 Vue/React/TypeScript 时，推荐前端开发工程师。
   - 技能包含 Python/FastAPI/Java/Go 时，推荐后端开发工程师。
   - 技能包含 SQL/数据时，推荐数据分析师。
   - 技能包含产品时，推荐产品经理。
   - 否则推荐应用开发工程师。
5. 前端构造 `CareerPlan` 结构并渲染结果区。

## 5. 未来后端对接建议

如果后端后续实现职业规划模块，建议按现有前端类型直接对接：

### 创建职业画像

`POST /api/career-planning/profiles`

请求体：

```json
{
  "education": "bachelor",
  "experience": "1-3",
  "skills": ["Vue.js", "TypeScript"],
  "work_description": "主要工作内容...",
  "weekly_learning_hours": 8,
  "preferred_target_role": "前端开发工程师",
  "projects": [
    {
      "name": "项目名称",
      "description": "项目描述",
      "role": "个人角色",
      "file_ids": [1001]
    }
  ]
}
```

响应体：

```json
{
  "id": 300,
  "education": "bachelor",
  "experience": "1-3",
  "skills": ["Vue.js", "TypeScript"],
  "work_description": "主要工作内容...",
  "weekly_learning_hours": 8,
  "preferred_target_role": "前端开发工程师",
  "projects": [],
  "created_at": "2026-07-15T00:00:00",
  "updated_at": "2026-07-15T00:00:00"
}
```

### 创建规划任务

`POST /api/career-plans`

请求体：

```json
{
  "profile_id": 300,
  "preferred_target_role": "前端开发工程师",
  "plan_months": 6
}
```

响应体：

```json
{
  "task_id": "task-xxx",
  "status": "pending",
  "result_id": null,
  "plan_id": 500,
  "poll_after_seconds": 2
}
```

### 轮询任务

`GET /api/ai/tasks/{task_id}`

复用现有 AI 任务接口，任务类型建议为 `career_plan`。

### 获取规划结果

`GET /api/career-plans/{plan_id}`

响应体应匹配前端 `CareerPlan` 类型，核心字段包括：

- `career_profile_summary`
- `recommended_roles`
- `career_goals`
- `skill_gap_analysis`
- `learning_path`
- `action_plan`
- `risks_and_alternatives`

### 项目附件

`POST /api/career-plans/project-files/upload`

表单字段：

- `file`: PDF、DOCX、TXT

响应体：

```json
{
  "id": 1001,
  "original_filename": "项目说明.pdf",
  "file_type": "pdf",
  "file_size": 102400,
  "status": "completed",
  "error_message": null
}
```

`DELETE /api/career-plans/project-files/{file_id}`

成功返回：

```json
null
```
