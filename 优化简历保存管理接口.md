# 优化简历保存管理接口文档

> 更新时间：2026-07-15  
> 前端模块：简历管理 / 我的简历 / 优化简历  
> 页面文件：
> - `src/views/resume/ResumeListView.vue`
> - `src/views/resume/SavedOptimizationDetailView.vue`
> - `src/views/resume/ResumeOptimizeView.vue`
> Base URL：`/api`

## 1. 功能说明

用户在 AI 简历优化页点击“保存更改内容”后，后端应保存一份优化后的简历版本。

前端已在“我的简历”页面新增标签页：

- 原始简历
- 优化简历

保存后的优化简历会展示在“我的简历 - 优化简历”中，用户可以：

- 查看优化简历详情
- 下载优化简历
- 删除优化简历

## 2. 保存更改内容

### 请求

`POST /api/resumes/{resume_id}/optimizations/{optimization_id}/save`

### 请求体

```json
{
  "optimized_content": "最终保存的优化简历内容"
}
```

### 后端处理建议

- 保存一条优化简历版本记录。
- 建议记录原始简历 ID：`resume_id`。
- 建议记录优化任务/优化结果 ID：`optimization_id`。
- 建议保存最终内容：`optimized_content`。
- 建议保存标题：可以用原简历标题加后缀，例如 `张三-优化简历`。
- 建议设置：
  - `is_saved = true`
  - `saved_at = 当前时间`

### 响应

前端会读取返回的优化结果并刷新当前页面内容。

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 12,
    "resume_id": 7,
    "title": "张三-优化简历",
    "is_saved": true,
    "saved_at": "2026-07-15T23:20:00",
    "optimization_summary": "本次优化重点是突出项目成果和岗位关键词。",
    "original": "原始简历内容",
    "optimized": "最终保存的优化简历内容",
    "optimized_content": "最终保存的优化简历内容",
    "score_improvement": 18,
    "change_items": [],
    "confirmation_questions": [],
    "created_at": "2026-07-15T23:10:00"
  }
}
```

## 3. 获取保存的优化简历列表

### 请求

`GET /api/resume-optimizations/saved`

### Query 参数

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `page` | number | 否 | 页码 |
| `page_size` | number | 否 | 每页数量 |

### 响应

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "items": [
      {
        "id": 12,
        "resume_id": 7,
        "title": "张三-优化简历",
        "is_saved": true,
        "saved_at": "2026-07-15T23:20:00",
        "optimization_summary": "本次优化重点是突出项目成果和岗位关键词。",
        "optimized_content": "最终保存的优化简历内容",
        "score_improvement": 18,
        "created_at": "2026-07-15T23:10:00"
      }
    ],
    "total": 1,
    "page": 1,
    "page_size": 20
  }
}
```

前端列表卡片会使用：

- `id`
- `resume_id`
- `title`
- `optimization_summary`
- `saved_at`
- `created_at`

## 5. 下载保存的优化简历

### 请求

`GET /api/resume-optimizations/saved/{saved_optimization_id}/download`

### 响应

返回文件流。

建议响应头：

```http
Content-Type: application/vnd.openxmlformats-officedocument.wordprocessingml.document
Content-Disposition: attachment; filename*=UTF-8''%E5%BC%A0%E4%B8%89-%E4%BC%98%E5%8C%96%E7%AE%80%E5%8E%86.docx
```

如果暂时不生成 DOCX，也可以返回 TXT 文件流，但建议文件名明确为优化简历。

## 6. 删除保存的优化简历

### 请求

`DELETE /api/resume-optimizations/saved/{saved_optimization_id}`

### 响应

```json
{
  "code": 0,
  "message": "success",
  "data": null
}
```

## 7. 前端路由

保存列表入口：

```text
/resume
```

页面中选择“优化简历”标签查看。

优化简历详情：

```text
/resume/optimized/{saved_optimization_id}
```

## 8. 前端按钮与接口对应

| 前端位置 | 操作 | API |
| --- | --- | --- |
| AI简历优化页 | 保存更改内容 | `POST /api/resumes/{resume_id}/optimizations/{optimization_id}/save` |
| 我的简历-优化简历 | 获取列表 | `GET /api/resume-optimizations/saved` |
| 优化简历卡片 | 查看详情 | `GET /api/resume-optimizations/saved/{saved_optimization_id}` |
| 优化简历卡片/详情页 | 下载 | `GET /api/resume-optimizations/saved/{saved_optimization_id}/download` |
| 优化简历卡片/详情页 | 删除 | `DELETE /api/resume-optimizations/saved/{saved_optimization_id}` |

## 9. 权限与数据归属建议

后端所有接口都应基于当前登录用户过滤数据：

- 用户只能看到自己保存的优化简历。
- 用户只能删除自己的优化简历。
- `saved_optimization_id` 不属于当前用户时，应返回 403 或 404。

推荐错误格式：

```json
{
  "detail": "优化简历不存在或无权限访问"
}
```
