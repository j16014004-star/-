# 职业生涯规划接口文档

本次前端已把职业生涯规划页面接入真实接口，不再使用本地模拟生成。后端按以下接口实现即可。

## 1. 上传项目附件

`POST /api/career-plans/project-files/upload`

请求：`multipart/form-data`

字段：

- `file`: PDF / DOCX / TXT 文件

返回 `data`：

```json
{
  "id": 401,
  "original_filename": "项目说明.pdf",
  "file_type": "pdf",
  "file_size": 123456,
  "status": "completed",
  "error_message": null
}
```

## 2. 删除项目附件

`DELETE /api/career-plans/project-files/{fileId}`

返回：

```json
{
  "code": 200,
  "message": "success",
  "data": null
}
```

## 3. 创建职业规划档案

`POST /api/career-planning/profiles`

请求：

```json
{
  "education": "bachelor",
  "experience": "1-3",
  "skills": ["Python", "FastAPI", "SQL"],
  "work_description": "主要工作经历描述",
  "weekly_learning_hours": 8,
  "preferred_target_role": "后端开发工程师",
  "projects": [
    {
      "name": "项目名称",
      "description": "项目描述",
      "role": "后端开发",
      "file_ids": [401]
    }
  ]
}
```

返回 `data`：在请求字段基础上增加 `id`、`created_at`、`updated_at`。

## 4. 创建职业规划生成任务

`POST /api/career-plans`

请求：

```json
{
  "profile_id": 301,
  "preferred_target_role": "后端开发工程师"
}
```

返回 `data`：

```json
{
  "task_id": "task_xxx",
  "status": "pending",
  "result_id": null,
  "plan_id": 501,
  "poll_after_seconds": 1
}
```

## 5. 查询 AI 任务进度

`GET /api/ai/tasks/{taskId}`

返回 `data`：

```json
{
  "task_id": "task_xxx",
  "task_type": "career_plan",
  "status": "generating",
  "progress": 45,
  "result_id": null,
  "error_message": null,
  "poll_after_seconds": 1
}
```

状态说明：`pending`、`preparing`、`generating`、`validating`、`saving`、`success`、`failed`、`cancelled`。

## 6. 获取职业规划结果

`GET /api/career-plans/{planId}`

返回 `data` 使用前端 `CareerPlan` 结构，至少包含：

- `career_profile_summary`
- `recommended_roles`
- `career_goals`
- `skill_gap_analysis`
- `learning_path`
- `action_plan`
- `risks_and_alternatives`
- `created_at`

前端成功轮询到 `success` 后，会使用 `task.result_id || createPlan.plan_id` 调用该接口。
