# 简历管理最新接口

> 更新时间：2026-07-15  
> 前端地址：`http://localhost:3000`  
> 后端地址：`http://localhost:8000`  
> 前端统一请求前缀：`/api`（开发环境由 Vite 代理到后端 8000 端口）

## 统一约定

- 除登录、注册外，以下接口都需要请求头：`Authorization: Bearer <access_token>`。
- 普通 JSON 接口统一返回：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

- FastAPI 业务错误通常返回 `detail` 字段，前端已兼容显示 `detail` 和 `message`。
- 简历资源由后端根据当前 JWT 用户限制所有权，前端不传用户 ID。
- 当前简历模块不提供“开始分析”功能，不得请求 `ai-analysis` 或 `ai-analyses` 路径。

## 1. 上传简历

```http
POST /api/resumes/upload
Content-Type: multipart/form-data
```

表单字段：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `file` | File | 是 | 支持 PDF、DOC、DOCX，默认最大 10MB |
| `title` | string | 否 | 简历标题 |

成功后 `data` 返回简历基础信息。前端必须使用后端返回的真实 `data.id` 跳转详情页，不生成临时 ID。

## 2. 获取我的简历

```http
GET /api/resumes
```

当前后端返回全部简历，暂不接收分页参数。

```json
{
  "items": [],
  "total": 0,
  "page": 1,
  "page_size": 0
}
```

## 3. 获取简历详情

```http
GET /api/resumes/{resume_id}
```

主要返回字段：

| 字段 | 说明 |
|---|---|
| `id` | 简历 ID |
| `title` | 标题 |
| `file_type` | `pdf`、`doc` 或 `docx` |
| `file_size` | 文件大小（字节） |
| `status` | `pending`、`processing`、`completed` 或 `failed` |
| `extracted_text` | 清洗后的完整文本 |
| `structured_data` | 规则结构化解析结果 |
| `chunks` | 文本分块，字段包含 `chunk_index`、`content` 等 |
| `error_message` | 处理失败原因 |

只有状态为 `completed` 且存在 `extracted_text` 时，前端启用“AI优化简历”按钮。

## 4. 下载原始简历

```http
GET /api/resumes/{resume_id}/download
```

返回文件流，不使用统一 JSON 包装。前端按 `Content-Disposition` 获取文件名。

## 5. 删除简历

```http
DELETE /api/resumes/{resume_id}
```

成功响应的 `data` 为 `null`。后端同时删除当前用户的简历记录、分块和受控目录中的原始文件。

## 6. 创建 AI 简历优化任务

```http
POST /api/resumes/{resume_id}/optimizations
Content-Type: application/json
```

请求体：

```json
{
  "optimization_type": "general",
  "target_role": null,
  "optimization_focus": ["all"],
  "style": "professional",
  "preserve_structure": true
}
```

字段约束：

- `optimization_type`：`general` 或 `target_role`。
- `target_role`：目标岗位优化时必填，通用优化时不发送。
- `optimization_focus`：可选 `work_experience`、`project_experience`、`skills`、`ats`、`all`；`all` 不能和其他项同时使用。
- `style`：`professional`、`technical`、`management` 或 `graduate`。
- `preserve_structure`：是否尽量保留原结构。

返回 `task_id`、`status`、`result_id` 和 `poll_after_seconds`。相同用户、简历和配置可能复用已有任务。

## 7. 查询 AI 任务状态

```http
GET /api/ai/tasks/{task_id}
```

任务状态：

```text
pending / preparing / generating / validating / saving /
success / failed / cancelled
```

前端按 `poll_after_seconds` 轮询。状态为 `success` 后使用 `result_id` 获取优化结果；`failed` 或 `cancelled` 时停止轮询并显示 `error_message`。

## 8. 获取 AI 优化结果

```http
GET /api/resumes/{resume_id}/optimizations/{optimization_id}
```

主要返回字段：

- `id`
- `optimization_summary`
- `original`
- `optimized`
- `optimized_content`
- `score_improvement`（当前允许为 `null`）
- `change_items`
- `confirmation_questions`
- `created_at`

## 已删除的旧接口

以下接口后端未实现，并且产品当前不需要简历分析功能，前端已删除相关调用：

```http
POST /api/resumes/{resume_id}/ai-analysis
GET /api/resumes/{resume_id}/ai-analyses/latest
GET /api/resumes/{resume_id}/ai-analyses
```

浏览器不应再出现类似下面的请求：

```text
http://localhost:3000/api/resumes/7/ai-analyses/latest
```
